Segmenti is a crop disease leaf segmentation engine designed to generate spatial disease segmentation masks for diagnostic analysis. It utilizes standard deep neural models to isolate and identify infected plant foliage sections.

## Architectural Features

*   **Pure PyTorch Architecture**: Decoupled from heavy, high-level trainer abstractions to allow granular control over gradients, state loading, and memory allocations.
*   **Dual-Architecture Factory**: Supports both **U-Net** and **DeepLabV3** networks, using ImageNet-pretrained backbones for stable convergence.
*   **Layer-Stratified Optimization**: Implements custom optimizer grouping to apply a lower learning rate to the backbone/encoder (preserving pretrained representations) and a standard learning rate to the segmentation heads.
*   **Error-Resilient Data Loading**: Built-in exception catchers prevent dataset file corruption from interrupting training by dynamically routing loader indices to safe fallback items.
*   **Smart State Loading**: Dynamic self-configuration reads runtime configurations directly from metadata files, enabling simple on-demand test inference.

## Getting Started

### Prerequisites

*   Python (v3.10+)
*   CUDA Toolkit (highly recommended for training operations)

### Installation

1. Clone the workspace:
```bash
git clone https://github.com/abderrahmenex86/segmenti.git
cd segmenti
```

2. Install the required libraries:
```bash
pip install -r requirements.txt
```

## Execution Workflows

### 1. Data Setup & Verification
Initialize structure and verify dataset alignment:
```bash
# Setup empty directory structures
python tools.py --mode download

# Align images and masks 
python tools.py --mode verify --root_dir dataset --dataset_name plantsegv3
```

### 2. Sanity Validation
Perform gradient flow checks, output dimensional verification, and verify loss calculation steps:
```bash
python main.py --mode test --root_dir dataset/plantsegv3 --num_classes 116
```

### 3. Hyperparameter Sweep
Perform an Optuna hyperparameter sweep (with dataset slicing for speed):
```bash
python main.py --mode optimize --root_dir dataset/plantsegv3 --num_classes 116 --optuna_trials 10
```

### 4. Training
Execute standard model training with automated metric checkpointing:
```bash
python main.py --mode train --root_dir dataset/plantsegv3 --model_type unet --num_classes 116 --epochs 40
```

### 5. Resuming Training
Recover and resume training from the latest save state:
```bash
python main.py --mode train --resume --num_classes 116
```

### 6. Inference
Run inference using a saved run folder's configuration and model weights:
```bash
python main.py --mode infer --image_path dataset/plantsegv3/images/val/sample_leaf.jpg --run_dir artifacts/20261024_153022_unet_116class
```

### 7. Performance Visualization
Render training metric curves:
```bash
python tools.py --mode plot
```
